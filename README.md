# Branch Rule

1. main branch
- develop 브랜치에서 문제가 없을 경우, 해당 브랜치에 최종 merge
- 일주일에 한번 develop에서 main으로 병합

2. develop branch
- 기본 브랜치에 가까운 main 브랜치에 올리기 전 merge하는 공간
- 평일에 한번씩 머지하는 공간은 이 공간으로 우리가 업데이트하는 브랜치

3. feature branch
- 영문 대문자 금지
- "-"(하이픈)만 사용
- 1기능 당 한개의 브랜치 생성
